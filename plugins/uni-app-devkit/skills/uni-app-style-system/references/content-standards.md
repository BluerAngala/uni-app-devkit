# 内容规范

本文件定义 uni-app 项目中的文案风格、数据格式和内容标准。

## 一、文案风格

### 语气

| 场景 | 正确 | 错误 |
|------|------|------|
| 成功提示 | 保存成功 | 恭喜您，数据已成功保存！！ |
| 错误提示 | 保存失败，请重试 | 出错了！！请联系管理员 |
| 空状态 | 暂无订单 | 抱歉，您还没有任何订单哦~ |
| 确认删除 | 确定删除吗？此操作不可恢复 | 温馨提示：您真的要删除这个宝贝吗？亲~ |
| 加载中 | 加载中... | 正在拼命加载中，请稍候... |

### 原则

- **简洁** — 能 5 个字说完的不用 10 个字
- **直接** — 说结果，不说过程
- **不卖萌** — 不用"哦~""呢""亲~""嘻嘻"
- **不甩锅** — 不用"请联系管理员""请联系客服"
- **不大写** — 不用"！！！""!!!"

### 禁用词

| 禁用 | 替代 |
|------|------|
| 恭喜/恭喜您 | 直接说结果 |
| 抱歉/对不起 | 直接说状态 |
| 温馨提示 | 直接说内容 |
| 亲/亲~ | 去掉 |
| 稍候/请稍候 | 加载中 |
| 联系管理员 | 请重试 / 展示具体错误 |

## 二、按钮文案

### 按钮文字长度

按钮文字控制在 **2-4 个字**，不要超过 6 个字。

| 场景 | 正确 | 错误 |
|------|------|------|
| 提交 | 提交 / 保存 | 点击此处提交表单 |
| 删除 | 删除 | 点击删除此条记录 |
| 新增 | 新增 / 添加 | 新增一条数据 |
| 搜索 | 搜索 | 点击搜索 |
| 取消 | 取消 | 取消本次操作 |

### 按钮文案规范

| 操作类型 | 文案 | 说明 |
|---------|------|------|
| 创建 | 新增 / 添加 / 创建 | 动词开头 |
| 编辑 | 编辑 / 修改 | 动词开头 |
| 删除 | 删除 | 单独使用 |
| 批量删除 | 批量删除 | 带数量提示 |
| 确认 | 确定 / 确认 | 不用"是" |
| 取消 | 取消 | 不用"否" |
| 搜索 | 搜索 | 不用"查询" |
| 导出 | 导出 / 下载 | 不用"导出数据" |
| 返回 | 返回 / 关闭 | 不用"返回上一页" |

## 三、数字格式

### 数字千分位

```js
// ✅ 1,234,567
// ❌ 1234567

function formatNumber(n) {
  return Number(n).toLocaleString('zh-CN')
}

// 或者手动
function formatNumber(n) {
  return String(n).replace(/\B(?=(\d{3})+(?!\d))/g, ',')
}
```

### 金额

```js
// ✅ ¥1,234.56
// ❌ 1234.56元
// ❌ RMB 1234.56

function formatMoney(cents) {
  return '¥' + (cents / 100).toFixed(2).replace(/\B(?=(\d{3})+(?!\d))/g, ',')
}

// 显示规则
// < 10000：显示精确数字 ¥1,234.56
// ≥ 10000：可简写 1.23万
// ≥ 100000000：可简写 1.23亿
```

### 百分比

```js
// ✅ 12.5%    （小数保留1位）
// ❌ 12.50%
// ❌ 12%

function formatPercent(n) {
  return (n * 100).toFixed(1) + '%'
}

// 增长/下降
// +12.5%   绿色
// -3.2%    红色
// 0%       灰色
```

### 大数字简写

```js
function formatCompact(n) {
  if (n >= 100000000) return (n / 100000000).toFixed(2) + '亿'
  if (n >= 10000) return (n / 10000).toFixed(2) + '万'
  return String(n).replace(/\B(?=(\d{3})+(?!\d))/g, ',')
}

// 1234       → 1,234
// 12345      → 1.23万
// 123456789  → 1.23亿
```

## 四、日期时间格式

### 格式选择

| 场景 | 格式 | 示例 |
|------|------|------|
| 列表日期 | YYYY-MM-DD | 2024-01-15 |
| 列表时间 | HH:mm | 14:30 |
| 详情完整时间 | YYYY-MM-DD HH:mm | 2024-01-15 14:30 |
| 精确到秒 | YYYY-MM-DD HH:mm:ss | 2024-01-15 14:30:00 |
| 今天 | 今天 HH:mm | 今天 14:30 |
| 昨天 | 昨天 HH:mm | 昨天 14:30 |
| 本年 | MM-DD HH:mm | 01-15 14:30 |
| 跨年 | YYYY-MM-DD | 2023-12-31 |

### 相对时间

```js
function relativeTime(timestamp) {
  const now = Date.now()
  const diff = now - timestamp
  const minute = 60 * 1000
  const hour = 60 * minute
  const day = 24 * hour

  if (diff < minute) return '刚刚'
  if (diff < hour) return Math.floor(diff / minute) + '分钟前'
  if (diff < day) return Math.floor(diff / hour) + '小时前'
  if (diff < 7 * day) return Math.floor(diff / day) + '天前'
  // 超过7天显示具体日期
  return formatDate(timestamp)
}
```

## 七、错误信息

### 错误信息格式

```
[对象] + [问题] + [建议]

✅ 商品名称不能为空
✅ 价格不能为负数
✅ 手机号格式不正确，请输入11位手机号
✅ 上传失败，请检查文件大小（不超过10MB）

❌ 错误
❌ 操作失败
❌ 参数错误
❌ 校验不通过
```

### 常见错误信息

| 场景 | 错误信息 |
|------|---------|
| 必填字段为空 | XX不能为空 |
| 格式错误 | XX格式不正确（如：手机号格式不正确） |

| 范围限制 | XX必须在N-M之间 |

| 网络断开 | 网络异常，请检查网络 |

| 未登录 | 请先登录 |
| 服务端错误 | 服务异常，请稍后重试 |
